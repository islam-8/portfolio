System Design

How It Works

A closed-loop control system — LDR sensors measure differential light intensity, Arduino Nano computes direction error, and servo motors actuate panel position to minimize that error continuously.

☀️
LDR Sensing
4 sensors measure ambient light intensity in each quadrant
→
🧮
Arduino Nano
Reads ADC values, computes differential error, generates PWM signals
→
⚙️
Servo Actuation
2× micro servos adjust panel azimuth and elevation axes
→
🔁
Closed Loop
Continuous monitoring and correction throughout the day
Problem
Fixed solar panels lose up to 40% efficiency due to sun angle drift
Solution
LDR differential sensing + dual-axis servo control loop
Result
Autonomous orientation toward peak irradiance all day

Control Logic

Tracking Algorithm

The Arduino Nano continuously runs a differential comparison loop at ~50 Hz to minimize the error between current panel angle and the sun's actual position.

1
LDR Data Acquisition — Arduino reads analog voltage from all 4 LDR voltage dividers via ADC channels A0–A3.
2
Differential Calculation — Computes horizontal error: (NW+SW) − (NE+SE) and vertical error: (NW+NE) − (SW+SE)
3
Dead-band Check — If error < threshold (noise margin), no movement issued — prevents jitter on cloudy/stable conditions.
4
Servo PWM Update — Arduino adjusts servo angles incrementally, writing new position to each servo channel until error = 0.
5
Continuous Loop — System repeats every ~20 ms, tracking sun movement across the sky throughout the day.
PSEUDO CODE
loop() {
  nw = analogRead(A0)
  ne = analogRead(A1)
  sw = analogRead(A2)
  se = analogRead(A3)

  errH = (nw+sw) - (ne+se)
  errV = (nw+ne) - (sw+se)

  if(abs(errH) > THRESHOLD)
    servoH.write(posH += step*sign(errH))
  if(abs(errV) > THRESHOLD)
    servoV.write(posV += step*sign(errV))

  delay(20)
}
⚡
Why This Works
LDR resistance drops as light increases. The differential reading tells the MCU which side gets more light — and thus which direction the sun is. The servo corrects until all four sensors read equal intensity — meaning the panel faces the sun directly.

Engineering Trade-offs
✓ Strengths
⚡Simple, robust design — minimal components, highly reliable
💰Very low BOM cost — accessible off-the-shelf components globally
🔁Truly autonomous closed-loop — no user intervention needed
🖨️3D printed chassis — fully customizable, repeatable manufacturing
⚠ Limitations
☁️LDR-based sensing affected by diffuse light and overcast conditions
🌙No GPS/astronomical model — relies on real-time optical feedback only
💨Light mechanical frame — not rated for outdoor wind/weather loads
📏Scale limited — servo torque constrains maximum panel size
